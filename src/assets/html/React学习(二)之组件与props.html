React学习(二)之组件与props

React

组件可以将UI切分成一些的独立的、可复用的部件，这样你就只需专注于构建每一个单独的部件。
组件从概念上看就像是函数，它可以接收任意的输入值（称之为“props”），并返回一个需要在页面上展示的React元素。

定义组件

方式一：工厂函数（无状态、最简洁）

  1. 1 function MyComponent1() { 
  2. 2 return <h2>工厂函数创建的</h2> 
  3. 3 } 

方式二：ES6类语法（复杂组件）

  1. 1 //这里必须extends React.Component及继承React.Component 
  2. 2 class MyComponent2 extends React.Component { 
  3. 3 render() { 
  4. 4 return <h3>ES6创建的</h3>; 
  5. 5 } 
  6. 6 } 

组件渲染

  1. 1 //这里注意，组件渲染引入需要用<ComponentName/>包裹 
  2. 2 ReactDOM.render(<MyComponent1/>, document.getElementById('example1')); 
  3. 3 ReactDOM.render(<MyComponent2/>, document.getElementById('example2')); 

注意：
1、组件的首字母需要大写，因为组件是以标签的形式引入，所以需要与HTML标签进行区分
2、虚拟DOM元素必须有只有一个根元素
3、虚拟DOM元素必须有结束标签

组件的三大属性之props

  1. 每个组件对象都会有props（properties的简写）属性
  2. 组件标签的所有属性都保存在props中
  3. 内部读取某个属性值：this.props.propertyName
  4. 作用：通过标签属性从组件外向组件内传递数据（单向）
  5. 对props中的属性值进行类型限制和必要性限制

基本用法

  1. 1 //定义组件 
  2. 2 class Person extends React.Component{ 
  3. 3 render(){ 
  4. 4 return ( 
  5. 5 <ul>

  7. 7 <li>性别：{this.props.sex}</li> 
  8. 8 <li>年龄：{this.props.age}</li> 
  9. 9 </ul> 
  10. 10 ) 
  11. 11 } 
  12. 12 } 
  13. 13 //设置组件的默认props属性值 
  14. 14 Person.defaultProps={ 
  15. 15 sex:'女' 
  16. 16 }; 
  17. 17 let person={username:'kobe', age:39, sex:'男'}; 
  18. 18 //渲染组件 
  19. 19 ReactDOM.render(<Person username={person.username} age={person.age}/>,document.getElementById('example1')); 
  20. 20 //注意这里没有传sex值，所以使用了默认设置的props值 

以上是props传值的基本用法，但是实际使用中可能会出现使用组件的人不知道哪些值需要，哪些值可以缺省。如果不传react也不会报错，这肯定是不行的；还有就是我们组建传值肯定是希望按照规则来进行（如：年龄值需要必须为Number类型），这种情况下就需要我们对传递的值进行验证。

PropTypes检查类型

  1. 1 import PropTypes from 'prop-types'; 
  2. 2 //设置props属性的必要性和传入的数据类型 
  3. 3 Person.propTypes = { 
  4. 4 username: PropTypes.string.isRequired, //isRequired为必填参数的意思 
  5. 5 age: PropTypes.number.isRequired, 
  6. 6 sex: PropTypes.string.isRequired 
  7. 7 }; 

以上是限定了Person组件props传参的条件（如：username必传且为字符串类型）
